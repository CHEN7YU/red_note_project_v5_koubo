🎬 脚本：

[HOOK - 前3秒]  
[lean in, lower voice] AI probably isn’t taking your boss’s job first.  
[beat, point down] It’s taking the bottom rung.  
[small shrug] Companies aren’t always firing seniors first. A lot of them are just… not opening junior roles.

[BODY - 主要内容]  
[fast pace] And this is bigger than one tool. Yeah, OpenAI’s Codex reportedly has 4 million weekly users, and around 20% aren’t even programmers.  
[raise finger] That non-programmer group is growing faster than the devs. So this is already leaking way beyond software engineers.

[cut, more animated] But it’s not just OpenAI. Claude Code, Copilot, Cursor, Devin, Gemini Code Assist — this whole AI coding and agent wave is changing who gets to build, and who gets paid to learn.  
[hand count] Stripe rolled out Claude Code to 1,370 engineers. Wiz used it to migrate 50,000 lines of Python to Go in like 20 hours — work they thought would take months.  
Rakuten cut new feature delivery from 24 working days to 5. Ramp says incident investigation time dropped 80%.  
[emphasis] And at Anthropic, they literally said most of their code is now written by Claude Code.

[slow down] That matters because the work AI is eating first?  
A lot of it is junior work. Drafting. Debugging. Cleanup. Migration. Research. Data pulls. First-pass analysis.  
[point to camera] The exact stuff people used to do to become mid-level.

[slightly wider shot] Microsoft looked at 20,000 AI users and mapped this shift: Author, Editor, Director, Orchestrator.  
Meaning: less doing every step yourself, more reviewing, steering, and managing multiple AI tools.  
[beat] So the job isn’t just “write code” anymore. It’s define the problem, judge the output, and run the system.

[TWIST/对比 - 转折点]  
[serious tone] And here’s the scary part: past tech waves usually created new entry-level paths. MIT researchers say AI might be different.  
[emphasis] Not because work disappears completely — but because the beginner stage gets compressed.

[SOLUTION - 建议]  
[energy up, practical tone] So what do you do if you’re early-career?  
Don’t wait to be trained. Start using AI as your copilot now.  
Build side projects with Claude Code, Cursor, Copilot — whatever you can access.  
[hand count] Use that to create a portfolio, not just a resume.  
Learn to be the person who frames the task, checks what’s wrong, and coordinates multiple tools.  
And honestly? AI also makes the one-person builder path way more real now. You don’t have to wait years for permission to start making stuff.

[CTA - 结尾]  
[look straight at camera] So yeah, the ladder is changing.  
But if junior jobs are shrinking, your move is to become really good at using AI before everyone else does.  
[point down] Are you still training to be an executor — or an orchestrator? Tell me in the comments.

## 🇨🇳 抖音中文版

🎬 脚本：

[HOOK - 前3秒]  
[身体前倾，压低声音] 这一轮 AI，可能还不是先抢领导的活。  
[停顿，加重] 它更像是先把职场最底下那一级台阶给拆了。  
[手一摊] 不是先裁掉所有老人，而是先不怎么招新人了。

[引入 - 背景交代]  
[像聊天一样] 这事真正扎心的，不是少几个岗位那么简单。  
而是以前那条“新人先练手、再升级、再独当一面”的路，现在可能正在被压缩。  
你想想，应届生刚进公司，最常干的是什么？  
写初稿、改文档、做整理、拉数据、写 SQL、修小 bug、跑流程、做第一版分析。  
这些活以前看着琐碎，但它其实是练级区。  
问题是，现在 AI 最先接过去的，偏偏就是这类活。

[干货1]  
[切换节奏，像讲重点] 而且这绝对不是 OpenAI 一家的事。  
是整个 AI 编程、AI Agent 工具的趋势都在往这边走。  
OpenAI 的 Codex，媒体报道周活已经到 400 万。  
里面大概 20% 还不是程序员，而且这批非程序员用户增长更快。  
这说明什么？  
说明“写代码”这件事，已经不再只属于程序员了。  
产品、运营、分析、创始人，甚至一些非技术团队，都开始直接用自然语言让 AI 帮自己做东西。

[干货2]  
[手指分层] 再看 Claude Code 这边的企业案例，就更夸张。  
Stripe 已经把 Claude Code 部署给了 1370 名工程师。  
有团队 4 天就完成了 1 万行 Scala 到 Java 的迁移，本来预估要 10 个工程师周。  
Wiz 用 Claude Code，20 小时迁了 5 万行 Python 到 Go，本来觉得要 2 到 3 个月。  
乐天把新功能交付时间，从 24 个工作日压到 5 天。  
Ramp 说，事故调查时间缩短了 80%。  
而且不只是工程师在用，销售、风控、财务这些非工程团队，也开始直接用自然语言查数据仓库，不用自己写 SQL。  
[加重] Anthropic 自己甚至说，现在“大部分代码都是 Claude Code 写的”。

[信息差/亮点]  
[表情严肃] 所以重点不是“哪家模型最强”。  
重点是：整个平台型工具、编程工具、Agent 工具一起上之后，很多原本给新人练手的工作，正在被整包吃掉。  
Copilot、Cursor、Devin、Gemini Code Assist，也都在把“一个人能做多少事”往上抬。  
那企业会怎么想？  
很简单：既然一个更成熟的人，加上 AI，能顶过去好几个人的产出，那我为什么还要像以前那样招一堆新人慢慢带？

[干货3]  
[手势比层级] 微软之前看了 2 万名 AI 用户，总结出四个阶段：Author、Editor、Director、Orchestrator。  
翻成人话就是：  
以前你是自己一点点干；  
后来变成先看 AI 做，再去改；  
再后来，你主要是给 AI 分配任务；  
最后，你是同时盯着好几个 AI 一起干。  
[停顿] 这就意味着，未来更值钱的能力，不只是执行。  
而是三件事：把问题定义清楚、判断结果靠不靠谱、再把多个 AI 工具编排起来。

[转折]  
[慢一点，压住情绪] 最麻烦的地方就在这儿。  
以前很多技术革命，都会顺手创造新的入门岗位。MIT 的研究大概就是这个意思。  
但 AI 可能不一样。  
它不是简单换个岗位名字让你进场。  
它可能是把“新人训练期”直接压短，甚至压没。  
[加重] 不是工作标题先没了，是成长路径先变窄了。

[SOLUTION - 建议]  
[语气转建设性，像认真给建议] 但也别听到这儿就觉得完了。  
真要说建议，我给你 4 句特别实在的。

第一，别等公司教你，马上开始把 AI 当副驾驶。  
你现在就去用 Claude Code、Cursor、Copilot，哪怕先从最小的项目开始。  
不是为了跟风，是因为以后不会用 AI 的新人，竞争力真的会更弱。

第二，自己给自己造“练级场”。  
既然企业不一定愿意给你足够多的练手机会，那你就自己做 side project。  
做个小工具、做个自动化流程、做个网站、做个数据分析项目。  
这些东西，就是你新的“实习经历”。

第三，别只训练自己当执行者，要训练自己当编排者。  
不要只会“把活干完”，要会“把问题说清楚、把标准定出来、把 AI 叫来一起干”。  
以后更值钱的人，往往不是最苦干的那个，而是最会调度的人。

第四，如果你本来就想做自己的东西，现在真的是一人公司最接近现实的时候。  
以前你得先去公司打几年工，攒一堆技能，才敢出来做产品。  
现在很多能力，AI 已经帮你补上了一大块。  
你可以更早开始做自己的产品、账号、服务，甚至小生意。

[CTA - 结尾]  
[看镜头，语气坚定] 所以结论不是“新人没路了”。  
而是以前那条老路，正在失效。  
新的路，是你更早学会和 AI 一起做事，更早拿作品说话，更早从执行者变成编排者。  
[手势指下方] 评论区告诉我，你现在最想先学的是哪一种：做项目、做作品集，还是学会管多个 AI？  
[微笑] 关注我，下一条我直接给你拆：普通人怎么用 AI 给自己补一段“消失的新人训练期”。

## 🔀 两版差异说明：
- Hook 不同在哪：
  - TikTok 用“AI 先拿走最底下那级台阶”做强冲击，直接切进“junior ladder”这个焦虑点，节奏更快。
  - 抖音用“新人训练期/新手村消失”去讲，更贴近大学生、应届生、转行人和 1 到 3 年职场人的体感。

- 举例不同在哪：
  - TikTok 版强调“这不是某一家公司”，快速扫过 Codex 非程序员增长 + Claude Code 企业案例 + Microsoft 框架，信息密度高但不展开太深。
  - 抖音版会把 Claude Code、Codex、Copilot、Cursor、Devin、Gemini 这些放进一个更完整的行业格局里，再把企业案例翻成更好懂的大白话。

- 结尾引导不同在哪：
  - TikTok 结尾偏个人定位，强调“别只做 executor，要变 orchestrator”，适合引发职业身份讨论。
  - 抖音结尾偏实操建议，明确给出 4 条路：现在就用 AI、自己做项目、练编排能力、尝试一人产品，让观众听完不至于只剩焦虑。

- 预测哪边效果更好：
  - 传播效率上，TikTok 版更容易靠“AI coding tools are changing the bottom rung”这种短句拿停留和转发。
  - 收藏和评论深度上，抖音版更强，因为它不只制造焦虑，还给了观众下一步能做什么。
